# JS 运行机制      词法环境、变量环境
- 变量环境
function 、var 声明的都在这里
- 词法环境
let const 
- 执行的代码
从上到下 顺序执行
- 上面三者都在执行上下文对象中
- 调用栈
 v8引擎用来管理函数之间的调用关系的一种数据结构
 - 编译总在发生执行前一刻
 - 全局和函数体的编译会生成执行上下文，存入调用栈
 - 当一个函数执行完毕后，他的执行上下文会弹出调用栈销毁  

 ## 编译的过程
 - 创建执行上下文对象
 - 找形参和变量声明，将形参和声明的变量名作为key，
 值为undefined
 - 统一形参和实参的值(全局没有这个步骤)
 - 找函数声明，将函数名作为key，值为函数体
 ## 总结
 执行上下文对象
 编译阶段， 先编译再执行
 js 执行流程
 读取代码 -> 编译 -> 执行
 调用栈
 负责管理函数之间的调用关系
 
## let const 
词法环境能够支持块级作用域
仍然是用栈管理不同作用域的变量
调用栈是执行上下文的容器
栈顶指针 指向当前正在执行的函数/全局
let const 不能重复声明，也不会变量提升，会进
入暂时性死区 dead zone 都是为之前JS 的bug 擦屁股
