# 轻松上手Vibe Coding 

想去用AI写代码， 但AI 生成的要不是 “幻觉代码” ， 看起来像那么回事， 一跑就出错。要么是屎山代码， 能跑， 但乱的想改都不知道从哪下手。

这个指南， 不是怎么写代码， 而叫我们去与AI 协作写出靠谱的代码。

- 新手 自然语言编程
- 不要急于写代码， 先思考，规划   md 文件
每次prompt 都会自动带上规划， 保持一致性， 有整体的约束 。

## 操作核心方法
- 第一步： 规划就是一切。
不要让来就让AI去写代码，先强制它去读懂你的技术栈和项目规划。(/init) 
就好像你不会让新来的员工直接干活， 去阿里入职 阿里Java 员工手册， 等，
得让它熟悉公司业务流程一样， AI 生成代码逻辑清晰多了
新手把codex , cc 工具， 把他们当伙伴，同事， 助手。 

## demo 
场景 用React + TailWind 实现一个任务清单(待办) 页面

vibe 新兵： 直接上手写代码 极易幻觉， 屎山。
    - 要不要本地存储
    - 要不要api 远程存储
    - 要不要拖拽排序 优先级别
    ....   
    帮我写一个React 待办清单页面， 支持新增、删除任务。

    - 先规划 ，再编码
```md
    遵守胶水编程思维： 优先使用成熟方案， 避免凭空造逻辑。
    第一个阶段： 只做规划， 禁止输出任何代码。
    1. 确认技术栈：
    React  19 + TailWindcss + useState 
    2. 梳理功能边界
    - 新增待办、删除待办、切换完成状态
    - 不做本地持久化、筛选、拖拽功能
    3.拆分模块(乐高积木)
    输入框组件、待办条目组件、列表容器组件、
    4. 定义数据流 
    useState 存储task 数组 数据结构：
    {id, text, completed}
    5. 输出这份完整规划， 等待我确认无误后， 再分段实现编码
```  

背后逻辑
先划定边界; 防止AI 擅自多加功能，无限膨胀成屎山代码。
强制模块拆分(好读，好维护，AI 生成的代码我们都要审核)，
引导AI复用独立组件，预先规定数据结构，从根源减少
AI 参数、字段幻觉 (title, 不能是text , 字段不能让AI 做主)。
先生成规划、审核规划、更新规划。
这份规划将伴随我们的项目开发完整周期。

第二步: "胶水"编程思维
能抄不写(github 经典， 优质， 经过验证的代码)， 能连不造(A、B、C 组件 胶水)。指挥AI 去找已经被验证千万次的成熟组件，积木拼装者，把他组装起来
胶水 比喻， 可以粘和， 少量（AI 生成的）， 产生幻觉和屎山代码的可能性大大降低。

胶水本身不创造零件， 只负责把现成零件黏在一起， 你只负责写衔接、调用、流转的粘合代码， 把各个模块联通。

轮子别人造好， 为我所用，我只做胶水；  胶水不生产零件， 只联通零件。
