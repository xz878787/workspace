## 思路
AI 工作流也能干活， 他和Agent（ReAct） 有何区别 ?
很多人会混淆， 反正都是AI 在执行任务

## Agent 智能体
和workflow 逻辑完全不一样的， 他不走在预设的轨道上， 而是站在一个开放的空间里， 自己去探索怎么去做(智能)。 从技术角度，一个Agent 往往具备三个大的核心能力

第一个能力: 感知环境 。
通过输入文本、图像、数据来理解任务， 有哪些工具可以调用， 当前情况如何。如何份步骤、如何规划。
第二个： 规划路径 
不是预先定义好， 而是动态去生成任务链。
第三个：执行行动
去调用工具， 完成子任务 ， 并不断的去调整。

## 举例
你让Agent 去规划旅行 ，他可能去搜一下机票和酒店的价格（感知环境），再去查天气和签证需求。然后结合你的偏好和预算， 动态去权衡方案。
整个过程， 他可能走的路径和你的预期完全不同， 这就是Agent 的关键区别， 自主性和适应性。他不会被一个固定的流程限制， 而是像一个思考中的个体。 本质的区别在于 : Workflow 是一个确定的执行， Agent 是一个不确定的探索， Workflow就像一条高速公路，
路线很清晰很固定， 效率很高， 但能到达你既定的终点。
Agent  更像一个司机， 他知道要去哪， 但是随时能变通   （感知环境），尝试新的路径， 甚至目的改变后还能改变 调整路径

所以， 从产业的角度， workflow 更适合流程化， 重复的任务。
比如审核流程  、数据清洗、客服AI 
Agent 更适合动态的一个角色场景。 比如: 复杂搜索、策略规划。

从技术栈的领域， workflow 更依赖于规则， 图形化编排（coze）， 流程引擎。 Agent 更依赖于大模型的能力，包括工具调用等。

未来我们的AI 工程中 会集合Agent 和 Workflow 的双重能力
底层是workflow 去提供稳定性和可控性， 上层时Agent 提供灵活性和智能型。

workflow 是骨架，Agent 是Workflow 的大脑。
workflow 没什么创造力， Agent 有， 但是不可控， 不可预期， 一起结合，既高效又有智慧 。
